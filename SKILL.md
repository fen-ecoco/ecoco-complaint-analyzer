# ECOCO 客訴分析與標籤小幫手 (ECOCO Complaint Analyzer)

此技能專門協助客服人員與數據分析師，自動判讀用戶的客訴意見（用戶內容），並將其精確歸類至對應的「問題類型」與「問題細項」。

## 什麼時候使用此技能

當使用者：
- 貼上一段或多段用戶在客服管道留下的意見、抱怨或問題描述（即「用戶內容」），要求給予分類建議時。
- 需要釐清某個客訴情境應該被歸類在何種「問題類型」與「問題細項」時。
- 希望批次分析多筆客訴資料並產出標籤報表時。

## 標籤分類準則 (Tagging Guidelines)

根據對照表，請使用以下的標籤維度進行分析與歸類：

### 1. APP使用問題類型
- **APP畫面顯示與機台狀態不符**
- **APP商家頁面空白**
- **APP點數顯示異常**
- **APP多重異常狀況**

### 2. APP帳號設定問題類型
- **忘記密碼/無法重設密碼**
- **帳號資訊修改/設定**
- **無法接收簡訊驗證碼**
- **APP無法登入**

### 3. 優惠券問題類型
- **兌換失敗/顯示錯誤**
- **無法進行兌換操作**
- **使用規則/限制條件說明**
- **查詢優惠券序號紀錄**

### 4. 回收點數問題類型
- **點數重複入點**
- **點數未入帳號**

### 5. 機台問題類型
- **機台運作中斷/重啟**
- **黑色分選門異常或卡瓶堵塞**
- **重量偵測異常**
- **操作流程異常/無法正常操作**
- **螢幕異常顯示/畫面異常**
- **履帶未作動或異常抖動**
- **機台當機/無回應**
- **機台需維護/故障提醒**
- **機台網路連線失敗**
- **機台髒污/需要清潔**
- **網路中斷或不穩定**
- **機台關閉/無法啟動**
- **投口綠燈拒收容器**
- **投入物卡住_瓶罐/電池**
- **辨識失敗異常或錯誤**
- **機台操作畫面無法登入**
- **投入後未獲點數/點數未記錄**
- **螢幕西曬導致黑屏或反光**
- **瓶蓋桶已滿**
- **回收艙門開啟**

### 6. 顧客關係類型
- **許願新增站點/設站建議**
- **申請刪除帳號**
- **更換帳號**
- **其他建議**
- **回收物使用規則**
- **相關活動規則疑問**

## 執行流程與判讀邏輯

當接收到用戶的「問題主旨」與「用戶內容」時，請綜合判讀這兩欄的語意，學習以下實際的客服分類邏輯範例：

### 【機台問題類型】判讀範例
- **機台關閉/無法啟動**: 
  - 主旨: 機器已關閉 / 內容: 新北市永和區國光路機器已關閉，請協助重啟
- **機台需維護/故障提醒**:
  - 主旨: 時常無法投遞 / 內容: 站點長期滿無法使用！超過一週都沒人來收走！
- **操作流程異常/無法正常操作**:
  - 主旨: 設備異常 / 內容: 畫面出現有殘液，鋁罐也不收
- **履帶未作動或異常抖動**:
  - 主旨: 機台故障，傳輸帶不會動 / 內容: 收瓶機履帶不轉動，無法回收
- **機台當機/無回應**:
  - 主旨: 十全店機器故障 / 內容: 寶特瓶放進去lag很久、螢幕無法觸控登入
- **瓶蓋桶已滿**:
  - 主旨: 滿了不能投 / 內容: 網路上顯示的機台只能投電池呢，都不能投寶特瓶

### 【APP帳號設定問題類型】判讀範例
- **無法接收簡訊驗證碼**:
  - 主旨: 收不到驗證碼 / 內容: 手機簡訊每個門號都收不到
- **APP無法登入**:
  - 主旨: 無法登入 / 內容: 不能登入看點數，一直跑出「發生未預期錯誤」
- **忘記密碼/無法重設密碼**:
  - 主旨: 無法收到認證碼 / 內容: 點選忘記密碼，卻一直無法收到認證碼

### 【回收點數問題類型】判讀範例
- **點數未入帳號**:
  - 主旨: 沒看到點數 / 內容: 點數沒入點，保特瓶也沒入點
  - 主旨: 點數未入個人帳戶 / 內容: 投入電池共計125點未入個人帳戶

### 【APP使用問題類型】判讀範例
- **APP點數顯示異常**:
  - 主旨: 點數不見了 / 內容: 點數還有200多點都不見了，會員資料也不見了
- **APP畫面顯示與機台狀態不符**:
  - 主旨: 回收站顯示可以投入，來時卻無法收 / 內容: 機台是可以收取的，實際操作卻顯示已滿不收

### 【優惠券問題類型】判讀範例
- **兌換失敗/顯示錯誤**:
  - 主旨: 家樂福折價券無法使用 / 內容: 尚未使用結帳成功卻顯示已使用
- **無法進行兌換操作**:
  - 主旨: 為何都無法兌換 / 內容: 我目前有2480點為何不能兌換

### 【顧客關係類型】判讀範例
- **其他建議**:
  - 主旨: 手錶掉落機台內 / 內容: 空瓶的時候，手錶表心不小心掉落與機台
- **相關活動規則疑問**:
  - 主旨: 加入會員 / 內容: 我只是要加入會員，開始回收而已

## 最終輸出格式要求

針對每一筆資料，請以清晰的格式回覆判定結果：

- **分析結果**：
  - 【問題類型】：對應的分類大項
  - 【問題細項】：對應的分類細項

如果在現有分類中找不到完全吻合的分類，請標註最接近的「問題類型」，並在「問題細項」處提供【建議新增標籤】供管理員參考。
