# ecoco-complaint-analyzer

ECOCO 客訴分析與標籤小幫手 (ECOCO Complaint Analyzer) 是一個可以幫助客服人員與數據分析師，自動判讀用戶的客訴意見（包含問題主旨與用戶內容），並將其精確歸類至對應的「問題類型」與「問題細項」的技能。

## 功能

此技能會進行語意綜合評斷字串，並依照 ECOCO 的真實客服情境分類法則進行映射分類，包含但不限於：

- ⚙️ **機台相關問題** (機台中斷重啟、網路連線失敗、瓶裝滿倉、辨識異常 等 45 項細項)
- 💳 **APP點數問題** (漏點、未入帳、重複入點 等 6 項細項)
- 🔒 **APP帳密登入問題** (簡訊驗證碼未收到、APP無法登入 等 9 項細項)
- 📱 **APP與機台連動問題** (畫面顯示與機台狀態不符 等 2 項細項)
- 🎟️ **App和合作商家問題** (兌換失敗、使用規則 等 6 項細項)
- ⚠️ **APP狀態問題** (多重異常狀況、頁面空白 等 4 項細項)
- 💡 **顧客關係類型** (設備設站建議、使用規則疑問 等 6 項細項)

## 什麼時候使用此技能

當使用者：
- 需要釐清某個客訴情境應該被歸類在何種「問題類型」與「問題細項」時。
- 希望批次分析多筆客訴資料 (包含 Excel 及 CSV 格式) 並產出標籤報表時。
- 貼上一段或多段用戶在客服管道留下的意見，要求給予分類建議時。

## 如何新增這項技能
使用 `skills` CLI 下載與執行：

```bash
npx skills add fen-ecoco/ecoco-complaint-analyzer
```

## 使用範例

**範例一：**
- **客訴主旨**：家樂福折價券無法使用
- **用戶內容**：尚未使用結帳成功卻顯示已使用
- **自動產製分析結果**：
  - 【問題類型】：優惠券問題類型
  - 【問題細項】：兌換失敗/顯示錯誤

**範例二：**
- **客訴主旨**：無法登入
- **用戶內容**：不能登入看點數，一直跑出「發生未預期錯誤」
- **自動產製分析結果**：
  - 【問題類型】：APP帳號設定問題類型
  - 【問題細項】：APP無法登入
